Password Security (AC-2)

Set as favorite Get notifications Share this item Provide feedback

Control Objective

All passwords must meet minimum security requirements

Controls

AC-2.1

Length

All passwords must have at least eight (8) characters.

AC-2.2

Complexity

All passwords must include at least one of each of the following: upper case, lower case, and non-alphabetic characters.

Example: bp$Y72=j
This is an example; do not use this password

AC-2.3

History

Password history must be a minimum of eight (8) passwords; Password history is a security setting that determines the number of unique passwords that have to be associated with a user account before the old password can be reused.

AC-2.5

Change After Reset

Passwords must be configured to require change at next login when reset by someone other than the end user, including automated processes.

AC-2.7

Unsuccessful Attempts

The number of consecutive attempts to enter an incorrect password must be limited to five (5) unsuccessful attempts, after which the User ID must be automatically locked until unlocked by a specific and authorized user initiated action.

AC-2.8

Disclosed Passwords

All passwords must be promptly changed if they are suspected of being disclosed, or are known to have been disclosed, to unauthorized parties.
