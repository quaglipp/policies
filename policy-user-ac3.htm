Entitlement Reviews (AC-3)

Requirement

User and system account entitlements reviews shall be conducted on a periodic basis based on the target system or data risk

Controls

AC-3.1

Scope
Access review scope must include the user account, the privilege, and the component or data being accessed.

Guideline: Last logon is not in scope for access reviews.
AC-3.2

Layers
Access reviews must be performed at all relevant solution layers.

Example: For a traditional system, access reviews must be performed at the application, server, and database levels.
AC-3.3

User Lists
Access review must be based on a system-generated user list.

Guideline: For solutions with user roles, the review process can be performed at the user role level in place of the user detailed account level.
AC-3.4

Reviewers
Access reviewers must not approve their own access.
AC-3.5

Frequency
Access reviews must be performed with the following minimum frequency:

AC-3.5.1.
Frequency: Subscriber Billing Systems
Access reviews for subscriber billing systems must be performed no less than monthly.
AC-3.5.2.
Frequency: Non-Subscriber Financial Systems
Access reviews for all other non-subscriber financial systems must be performed no less than quarterly.
AC-3.5.3.
Frequency: All Other Information Systems
Access reviews for all other information systems must be performed no less than annually.
AC-3.6

Partner and Vendor Reviews
Access reviews for users accounts associated with business partners and vendors must be performed every six (6) months
AC-3.7

Inactive Account Disablement
User accounts inactive for 90 days must be disabled
AC-3.8

Inactive Account Deletion
User accounts inactive for 180 days must be deleted
