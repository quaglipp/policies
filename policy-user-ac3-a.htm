Entitlement Reviews (AC-3)

l Active

 

Requirement

 

User and system account entitlements reviews shall be conducted on a periodic basis based on the target system or data risk

 

Standards

AC-3.1

 Access review scope must include the user account, the privilege, and the component or data being accessed.

SUBMIT / VIEW FEEDBACK FOR THIS STANDARD

 

Guideline: Last logon is not in scope for access reviews.

 

 

AC-3.2 Access reviews must be performed at all relevant solution layers.

 

SUBMIT / VIEW FEEDBACK FOR THIS STANDARD

 

Example: For a traditional system, access reviews must be performed at the application, server, and database levels.

 

 

AC-3.3 Access review must be based on a system-generated user list.

 

SUBMIT / VIEW FEEDBACK FOR THIS STANDARD

 

Guideline: For solutions with user roles, the review process can be performed at the user role level in place of the user detailed account level.

 

 

AC-3.4 Access reviewers must not approve their own access.

 

SUBMIT / VIEW FEEDBACK FOR THIS STANDARD

 

 

AC-3.5 Access reviews must be performed with the following minimum frequency:

Subscriber billing – monthly

All other financials – quarterly

All other information systems – annually

 

SUBMIT / VIEW FEEDBACK FOR THIS STANDARD

 

 

AC-3.6 Access reviews for users accounts associated with business partners and vendors must be performed every six (6) months

 

SUBMIT / VIEW FEEDBACK FOR THIS STANDARD

 

 

AC-3.7 User accounts inactive for 90 days must be disabled

 

SUBMIT / VIEW FEEDBACK FOR THIS STANDARD

 

 

AC-3.8 User accounts inactive for 180 days must be deleted.

 

SUBMIT / VIEW FEEDBACK FOR THIS STANDARD
